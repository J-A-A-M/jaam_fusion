    Яскравість¶

    Секція "Яскравість" керує тим, наскільки яскраво світитимуть LED у різних режимах і для різних подій.

    Важливо розуміти, що в прошивці є два “шари” яскравості:

    1. Поточна (загальна) яскравість — визначається режимом (ручний / день-ніч / сенсор) і зберігається як CURRENT_BRIGHTNESS.
    2. Локальна яскравість подій — повзунки типу "Тривога", "Без тривоги", "Домашній регіон" тощо.

    Фактична яскравість LED для події обчислюється як добуток загальної та локальної (в відсотках).

    Режим яскравості¶

    Перейдіть: Яскравість → Режим яскравості

    Доступні режими:

    • Ручний — використовує повзунок "Загальна"
    • День/Ніч — автоматично перемикає "День" та "Ніч" за розкладом
    • Сенсор освітлення — автоматично перемикає "День" та "Ніч" за рівнем освітлення (якщо сенсор доступний)

    Ручний¶

    Перейдіть: Яскравість → Загальна

    У цьому режимі яскравість встановлюється вручну і не залежить ані від часу доби, ані від сенсора.

    День/Ніч (за часом)¶

    Перейдіть:

    • Яскравість → День
    • Яскравість → Ніч

    Прошивка визначає “ніч” за розкладом, який задається у секції Загальні:

    • Загальні → Початок дня
    • Загальні → Початок ночі

    Сенсор освітлення¶

    Перейдіть:

    • Яскравість → День
    • Яскравість → Ніч
    • Яскравість → Рівень освітлення для застосування "нічної" яскравості

    Якщо сенсор освітлення доступний, прошивка порівнює поточний рівень освітленості (lux) з порогом:

    • якщо lightLevel < threshold → використовується "Ніч"
    • інакше → використовується "День"

    Note

    Якщо сенсор не підключений/недоступний або дані некоректні, прошивка повертається до базової логіки (фактично — до ручної “Загальної”).

    Примусовий “нічний” режим¶

    У прошивці існує прапорець примусової нічної яскравості (night mode), який має пріоритет над усіма режимами яскравості.

    Коли він увімкнений — завжди використовується значення "Ніч" незалежно від обраного режиму.

    Info

    Цей режим зазвичай вмикають кнопкою (дія "Увімк./Вимк. режим нічної яскравості" у секції "Апаратне забезпечення" → налаштування Long Click). Він не вимикається автоматично.

    Яскравість для подій та елементів¶

    У цій секції є повзунки для різних станів/подій (0–100%). Вони є локальними множниками.

    Базові¶

    • Тривога — яскравість для стану “тривога”
    • Без тривоги — яскравість для нормального стану
    • Домашній регіон — яскравість підсвічування домашнього регіону

    Загрози (за наявності)¶

    Повзунки для загроз показуються тільки якщо відповідні загрози увімкнені в секції "Тривоги":

    • Вибухи
    • Загроза ударних БПЛА
    • Розвідувальні БПЛА
    • Загроза крилатих та авіаційних ракет
    • Загроза КАБ
    • Загроза балістичних ракет

    Канали/режими¶

    • Фонова стрічка — яскравість фонового каналу
    • Режим лампи — яскравість у режимі лампи
    • Сервісні діоди — яскравість сервісної стрічки (індикаторів)

    Кінцева яскравість анімацій¶

    Повзунок "Кінцева яскравість анімацій" задає, до якої яскравості “дотухає” анімація наприкінці (використовується як endBrightness).

    Як 0–100% перетворюється у реальну яскравість LED¶

    Значення повзунків — це відсотки. Прошивка перетворює їх у фактичну яскравість (0–255) з урахуванням апаратного профілю:

    • Для кожного профілю є мінімальна та максимальна допустима яскравість.
    • 1–100% лінійно масштабується в цей діапазон.

    Tip

    Якщо на малих значеннях є мерехтіння — перевірте Апаратне забезпечення → Мінімальна яскравість. Якщо хочете обмежити “верх” — див. Апаратне забезпечення → Максимальна яскравість (з підтвердженням ризиків).

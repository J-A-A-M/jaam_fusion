# Файли даних карт регіонів

Ця тека містить вихідні дані для конфігурацій LED-маппінгу регіонів, що використовуються у прошивці JAAM Fusion.

## Файли

### `region_maps.json`
Головний файл з усіма конфігураціями карт регіонів у форматі JSON. Це основний файл даних, який потрібно редагувати для зміни маппінгу LED на регіони.

**Структура:**
```json
{
  "maps": {
    "REGION_MAP_JAAM_3_0": {
      "description": "Карта регіонів для JAAM 3.0 (548 LEDs, 169 регіонів)",
      "regions": [
        {
          "regionId": 9999,
          "name": "АР Крим",
          "leds": [14, 15, 16, ...]
        },
        ...
      ]
    },
    ...
  }
}
```

**Конфігурації карт:**
- `REGION_MAP_JAAM_3_0` - Плата JAAM v3.0 (169 регіонів, 548 LED)
- `REGION_MAP_JAAM_3_2` - Плата JAAM v3.2 (169 регіонів, 808 LED)
- `STATE_MAP_LED_ODESA_WITH_KYIV` - Одеська карта з Києвом (167 регіонів, 167 LED)
- `STATE_MAP_LED_ODESA_WITHOUT_KYIV` - Одеська карта без Києва (166 регіонів, 166 LED)
- `STATE_MAP_LED_TRANSCARPATHIA_WITH_KYIV` - Закарпатська карта з Києвом (167 регіонів, 167 LED)
- `STATE_MAP_LED_TRANSCARPATHIA_WITHOUT_KYIV` - Закарпатська карта без Києва (166 регіонів, 166 LED)

## Робочий процес

### Редагування карт регіонів

1. **Відредагуйте вихідні дані:**
   ```bash
   # Відкрийте та відредагуйте JSON файл
   code data/region_maps.json
   ```

2. **Регенеруйте C++ заголовок:**
   ```bash
   # Запустіть скрипт конвертації
   python3 tools/convert_region_map.py
   ```
   
   Або з власними шляхами:
   ```bash
   python3 tools/convert_region_map.py data/region_maps.json src/JaamConfig_Generated.h
   ```

3. **Перебудуйте прошивку:**
   ```bash
   platformio run
   ```
   
   **Примітка:** PlatformIO автоматично запустить скрипт конвертації перед збіркою завдяки `extra_scripts` у `platformio.ini`.

### Додавання нової конфігурації карти

1. Додайте новий запис карти до `region_maps.json`:
   ```json
   "НОВА_КАРТА": {
     "description": "Опис нової карти",
     "regions": [
       {"regionId": 1, "name": "Регіон 1", "leds": [0, 1, 2]},
       {"regionId": 2, "name": "Регіон 2", "leds": [3, 4, 5]},
       ...
     ]
   }
   ```

2. Запустіть скрипт конвертації для генерації C++ коду
3. Оновіть свій код для використання нових констант карти

## Примітки щодо формату даних

- **regionId**: Унікальний ідентифікатор кожного регіону (відповідає масиву DISTRICTS у JaamConfig.h)
- **name**: Читабельна назва регіону (з масиву DISTRICTS)
- **leds**: Масив індексів LED, призначених цьому регіону
- Спеціальні ID регіонів (155, 225, 332 тощо) представляють адміністративні одиниці зі спеціальною обробкою
- Region ID 9999 зазвичай використовується для не-регіональних LED (як індикатори статусу)

## Оптимізація пам'яті

Скрипт конвертації генерує оптимізовані структури плоских масивів:
- **Масив метаданих**: зберігає region_id, start_offset та led_count
- **Масив позицій LED**: плоский масив усіх індексів LED по порядку
- Ця структура економить ~15 КБ порівняно зі старим підходом map-per-region

## Згенеровані файли

Скрипт конвертації генерує:
- `src/JaamConfig_Generated.h` - C++ заголовок з усіма даними карт

**НЕ РЕДАГУЙТЕ** згенерований заголовок вручну - завжди редагуйте JSON файл і регенеруйте.

## Інструменти

### `tools/convert_region_map.py`
Python скрипт, що конвертує JSON дані у формат C++ заголовку.

**Використання:**
```bash
python3 tools/convert_region_map.py [input.json] [output.h]
```

**Вивід скрипта:**
```

============================================================

🗺️  JAAM_3_0
   Regions:     169
   LEDs:         548
   Metadata:      845 bytes
   LED array:    1096 bytes
   Total:        1941 bytes
   Savings:     ~15000 bytes vs old structure

...

============================================================
✅ Generated: src/JaamConfig_Generated.h

📊 Overall statistics:
   Maps:          6
   Regions:     1004 (avg 167 per map)
   LEDs:        2022 total
   Memory used:  9064 bytes
============================================================
```

**Вимоги:**
- Python 3.6+
- Без зовнішніх залежностей (використовує лише стандартну бібліотеку)

**Особливості генерації:**
- Автоматично додає коментарі з назвами регіонів до метаданих
- Коментарі з назвами регіонів перед кожним масивом LED
- Форматування single-line для масивів LED
- Детальна статистика використання пам'яті для кожної карти

### `tools/pre_build.py`
Unified pre-build скрипт для PlatformIO, що послідовно запускає:
1. `compress_assets.py` - стиснення веб-ресурсів
2. `convert_region_map.py` - генерація C++ заголовку з JSON

Автоматично викликається PlatformIO через `extra_scripts = pre:tools/pre_build.py` у `platformio.ini`.

**Не потрібно запускати вручну** - виконується автоматично при кожній збірці.

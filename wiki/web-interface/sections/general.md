# Загальні

Секція **"Загальні"** містить базові налаштування, з яких зазвичай починають після підключення пристрою до WiFi: вибір апаратного профілю, домашнього регіону, режиму мапи, часових параметрів, а також дії для WiFi та керування налаштуваннями.

## Режим прошивки

Перейдіть: **Загальні → Режим прошивки**

Це ключове налаштування, яке визначає апаратний профіль пристрою (плата/розкладка) і впливає на доступні опції в інтерфейсі.

У списку можна побачити (залежно від версії прошивки та складання):

- **Плата JAAM 1.3**
- **Плата JAAM 2.1**
- **Плата JAAM 3.0**
- **Плата JAAM 3.2**
- **Плата JAAM 3.2 (по областям)**
- **Початок на Закарпатті** / **Початок на Закарпатті + Київ**
- **Початок на Одещині** / **Початок на Одещині + Київ**
- **Власна карта LED**

### Редактор мапи

Кнопка **"Редактор мапи"** з’являється лише тоді, коли обрано **"Власна карта LED"**.

Вона відкриває редактор, де можна створити власне зіставлення LED ↔ регіони.

- Детальніше: [Редактор мапи](../map-editor.md)
- Загальний опис LED-мапінгу: [LED Mapping](../../hardware/led-mapping.md)

## Домашній регіон

Перейдіть: **Загальні → Домашній регіон**

Домашній регіон використовується як “ваш” регіон для сповіщень і логіки відображення (зокрема для подій, звуку/сирени та підсвічування домашнього регіону).

!!! tip
    За замовчуванням домашній регіон встановлено на **Київ**.

## Режим фонової стрічки

Перейдіть: **Загальні → Режим фонової стрічки**

Це налаштування доступне не на всіх апаратних профілях. Воно визначає, як саме працює **фонова LED-стрічка** (якщо вона підключена у вашій конфігурації).

Доступні режими:

- **Домашній Регіон** — фон підлаштовується під логіку відображення та ваш домашній регіон
- **Власний колір** — фон використовує один обраний колір
- **Індивідуальні кольори** — фон використовує карту кольорів (колір задається для кожного регіону)

!!! note
    Колір для режиму **"Власний колір"** задається в секції **"Анімації"** (параметр **"Задня підсвітка"**).

### Редактор кольорів

Кнопка **"Редактор кольорів"** з’являється, коли обрано **"Індивідуальні кольори"**.

У редакторі можна задати кольори для фонового відображення по регіонах.

- Детальніше: [Редактор кольорів](../color-editor.md)

## Режим мапи

Перейдіть: **Загальні → Режим мапи**

Це налаштування визначає, **що саме показує мапа** (основна стрічка) у поточний момент.

Доступні режими (як у списку прошивки):

- **Вимкнено** — мапа вимкнена
- **Тривога** — відображення тривог/подій
- **Погода** — відображення температури по регіонах
- **Прапор** — режим прапора
- **Випадкові кольори** — декоративний режим
- **Лампа** — режим настільної лампи

Також у деяких конфігураціях можуть бути доступні додаткові режими:

- **Енергосистема**
- **Радіація**

## Налаштування "Нічного режиму"

У блоці **"Налаштування \"Нічного режиму\""** задається час початку дня і ночі:

- **Початок дня**
- **Початок ночі**

Ці параметри використовуються лише якщо відповідний режим увімкнено в секціях **"Яскравість"**, **"Звук"** та/або **"Дисплей"**.

!!! info
    Типові значення за замовчуванням: день — **08:00**, ніч — **22:00**.

## Хвилина мовчання

Параметр **"Увімкнути режим \"Хвилина мовчання\" о 9:00"** керує тим, чи застосовувати спеціальний режим щодня о 09:00.

## Логи в Serial

Параметр **"Виводити логи в Serial"** вмикає детальні логи через Serial для діагностики.

!!! warning
    Увімкнення Serial-логів може знижувати продуктивність. На показ логів у панелі **"Логи пристрою"** у веб-інтерфейсі це не впливає.

## Часовий пояс

Перейдіть: **Загальні → Часовий пояс**

Тут можна вибрати часовий пояс (наприклад, **Europe/Kyiv (UTC+2)**). Часовий пояс впливає на:

- відображення часу
- роботу розкладу дня/ночі
- події на кшталт **"Хвилини мовчання"**

## Назва та опис пристрою

- **Назва пристрою** — відображається в інтерфейсі та використовується як назва (instance name) сервісів у mDNS
- **Опис пристрою** — текстовий опис для інтерфейсу/ідентифікації

!!! note
    Адреса доступу виду `http://jaam_xxxxxx.local` визначається мережевим hostname (mDNS hostname) і може відрізнятися від **"Назви пристрою"**.

## Керування налаштуваннями

У блоці **"Керування налаштуваннями"** є дії:

- **Зберегти** — створити резервну копію налаштувань
- **Відновити** — відновити налаштування з резервної копії
- **Скинути** — скинути налаштування

!!! note
    Налаштування WiFi не змінюються цими діями (як зазначено в інтерфейсі).

## Конфігурація WiFi

Кнопка **"Налаштування WiFi"** відкриває сторінку `/wifi` для керування збереженими WiFi мережами: скан, додавання та видалення мереж.

Детальніше: [Мережа → Налаштування WiFi](network.md).

# Анімації

Секція **"Анімації"** задає:

- **тип анімації** для початку/відбою тривоги та для окремих загроз
- **тривалість показу** (скільки секунд анімація має програватися)
- **період циклу** (швидкість: як часто повторюється один “крок” анімації)
- **кольори** для подій та елементів (домашній регіон/фон/лампа)

Більшість параметрів працює для режиму мапи **"Тривоги"**.

!!! note
    Частина подій (дрони/ракети/КАБ/балістика/вибухи) відображається тільки якщо їх увімкнено в секції **"Тривоги"** — тоді ж у секції **"Анімації"** з’являються відповідні налаштування.

## Синхронні анімації

Перейдіть: **Анімації → Синхронні анімації**

Якщо увімкнено, прошивка синхронізує фазу анімації для одного й того самого типу (наприклад, “мерехтіння” для тривоги) — тобто різні LED/регіони “пульсують” в одному ритмі.

Якщо вимкнено, анімація починається “з нуля” в момент події для кожного регіону окремо, тому ефект може бути менш рівномірним.

## Попередній перегляд анімацій

Перейдіть: **Анімації → Попередній перегляд анімацій**

Коли опцію увімкнено, при зміні таких параметрів:

- типу анімації
- періоду циклу
- кольору

…прошивка запускає короткий **preview** на стрічці, щоб одразу було видно результат.

!!! tip
    Preview використовує поточну яскравість відповідної події (див. секцію **"Яскравість"**), тому якщо здається “занадто тьмяно/яскраво” — перевірте яскравість для цієї події.

## Типи анімацій для подій

Перейдіть:

- **Анімації → Початок тривог**
- **Анімації → Відбій тривог**
- (за наявності) **Загроза ударних БПЛА / Розвідувальні БПЛА / Загроза крилатих та авіаційних ракет / Загроза КАБ / Загроза балістичних ракет / Вибухи**

У кожної події можна вибрати тип анімації зі списку **animation_types**. Зазвичай доступні такі типи:

- **Без анімації**
- **Циклічне затухання**
- **Мерехтіння**
- **Кольорове мерехтіння**
- **Перехід між кольорами**
- **Односторонній перехід між кольорами**
- **Пульсація**
- **Кольорова пульсація**

## Тривалість показу (секунди)

Цей блок підписаний як **"Налаштування тривалості показу анімацій (в секундах)"**.

Перейдіть до відповідного повзунка для події, наприклад:

- **Анімації → Початок тривог** (час)
- **Анімації → Відбій тривог** (час)
- …і т.д.

Діапазон у UI: **5…600 сек** (крок **5**).

Це “загальний час”, протягом якого анімація буде активною після події.

## Період одного циклу (мілісекунди)

Цей блок підписаний як **"Налаштування тривалості одного циклу анімації (в мілісекундах)"**.

Перейдіть до відповідного повзунка для події, наприклад:

- **Анімації → Початок тривог** (цикл)
- **Анімації → Відбій тривог** (цикл)

Діапазон у UI: **300…5000 мс** (крок **100**).

Менший період = швидша анімація (частіше мерехтить/пульсує).

### Як тривалість пов’язана з циклом

У прошивці кількість повторень (“cycles”) фактично розраховується як:

$$\text{cycles} = \frac{\text{timeSeconds} \cdot 1000}{\text{cyclePeriodMs}}$$

Тобто:

- **time** визначає “скільки довго”
- **cycle** визначає “наскільки швидко”

!!! warning
    Надто малі значення можуть виглядати як часте миготіння.

## Налаштування кольорів

Цей блок підписаний як **"Налаштування кольорів"**.

Тут задаються базові кольори, які потім використовуються в анімаціях і відображенні:

- **Початок тривог**
- **Відбій тривог**
- (за наявності) кольори для окремих загроз
- **Домашній регіон**
- **Задня підсвітка**
- **Режим лампи**

!!! note
    Кольори для загроз показуються лише якщо відповідна загроза увімкнена в секції **"Тривоги"**.

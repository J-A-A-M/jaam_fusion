# Troubleshooting

Рішення поширених проблем з JAAM Fusion.

## Проблеми з підключенням

### Пристрій не створює WiFi мережу

**Можливі причини:**
- Недостатнє живлення
- Прошивка не встановилася коректно
- Несправна плата ESP32

**Рішення:**

1. Перевірте живлення (має бути стабільні 5V/500mA+)
2. Прошийте пристрій ще раз через [FLASHER](https://flasher.jaam.net.ua/)
3. Переконайтеся що світлодіод на платі блимає при завантаженні
4. Спробуйте іншу плату ESP32

### Не можу підключитися до веб-інтерфейсу

**Перевірте:**

```
1. ESP32 та комп'ютер в одній мережі
2. IP-адреса правильна (подивіться в роутері)
3. Firewall не блокує підключення
4. Порт 80 відкритий
```

**Спробуйте:**

- `http://jaam_xxxxxx.local` (mDNS, де XXXXXX - та ж частина MAC адреси)
- `http://192.168.1.XXX` (IP з роутера)
- Скиньте налаштування WiFi (Boot кнопка 5 сек)

## Проблеми з LED

### LED не світяться взагалі

**Чеклист:**

- [ ] LED стрічка підключена до правильного піна (за замовчуванням GPIO2)
- [ ] LED стрічка має живлення (окремий БЖ для >10 LED)
- [ ] Data pin підключений до Data In (не Data Out!)
- [ ] Спільний GND між ESP32 та БЖ LED

**Перевірте в налаштуваннях:**

```
Settings → Hardware → LED Pin: 2
Settings → Hardware → LED Count: [ваша кількість]
```

### LED світяться неправильними кольорами

**Рішення:**

Змініть тип LED:

```
Settings → Hardware → LED Type
```

Варіанти:
- `WS2812B` - порядок GRB (найпоширеніший)
- `WS2811` - порядок RGB
- `SK6812` - порядок GRBW (з білим)

### Мерехтіння LED

**Причини:**

1. Недостатнє живлення
2. Довгі дроти до LED
3. Нестабільний сигнал

**Рішення:**

1. Використовуйте потужніший БЖ
2. Додайте конденсатор 100-470мкФ біля живлення LED
3. Додайте резистор 220-470 Ом між Data pin та LED
4. Використовуйте екрановані дроти

## Проблеми з OLED дисплеєм

### OLED не показує нічого

**Перевірте:**

```
1. Правильні I2C піни (SDA/SCL)
2. I2C адреса (0x3C або 0x3D)
3. Напругу живлення (3.3V або 5V залежно від дисплею)
```

**Команда для перевірки I2C:**

Увімкніть режим debug в налаштуваннях та подивіться в Serial Monitor чи виявляється дисплей.

## Проблеми з даними

### Дані не оновлюються

**Перевірте:**

1. Інтернет-з'єднання
2. Статус серверу [jaam.net.ua](http://jaam.net.ua/)
3. Час на пристрої (має бути синхронізований)

```
Settings → General → NTP Sync: Enabled
```

### Неправильний час

**Рішення:**

```
Settings → General → Timezone: Europe/Kyiv
Settings → General → NTP Server: pool.ntp.org
```

Примусово синхронізувати:

```
Settings → System → Sync Time Now
```

## Проблеми з продуктивністю

### Пристрій перезавантажується

**Можливі причини:**

- Недостатньо пам'яті
- Watchdog timeout
- Нестабільне живлення

**Перевірте логи:**

```
Web Interface → System → Logs
```

**Рішення:**

1. Зменшіть кількість LED
2. Збільште watchdog timeout
3. Покращіть живлення

### Повільний веб-інтерфейс

**Рішення:**

1. Очистіть кеш браузера
2. Використовуйте сучасний браузер (Chrome/Firefox)
3. Перезавантажте ESP32

## Скидання налаштувань

### Повний скид до заводських

**Метод 1: Через веб-інтерфейс**

```
Settings → System → Factory Reset
```

**Метод 2: Кнопка Boot**

Затисніть кнопку Boot на 10 секунд до мигання LED.

**Метод 3: Через Serial**

Підключіться через Serial Monitor та відправте команду:

```
factoryreset
```

## Отримання діагностики

Для звернення в підтримку зберіть наступну інформацію:

```
- Версія прошивки (Settings → About)
- Модель ESP32
- Логи (Web Interface → System → Logs)
- Конфігурація LED (Map Editor → Export)
- Скріншот проблеми
```

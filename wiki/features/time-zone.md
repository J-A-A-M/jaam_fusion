# Часовий пояс і літній/зимовий час

У JAAM Fusion час потрібен не лише для годинника на дисплеї, а й для подій і розкладів (день/ніч, “Хвилина мовчання” тощо). Тому важливо правильно обрати **часовий пояс**.

## Як вибрати часовий пояс

Перейдіть: **Загальні → Часовий пояс**

Після вибору прошивка застосовує:

- зсув від UTC (години/хвилини)
- правила переходу на літній/зимовий час (якщо вони передбачені для цього поясу)

Детальніше про секцію: [Загальні](../web-interface/sections/general.md)

## Автоматичний перехід на літній/зимовий час

Для частини часових поясів прошивка вмикає **автоматичне DST** (Daylight Saving Time) — це і є перехід на літній/зимовий час.

- якщо для обраного поясу DST використовується — прошивка застосовує його автоматично
- якщо для обраного поясу DST не використовується — автоматичного переходу не буде

!!! note
    Правила DST залежать від обраного часового поясу. У прошивці вони задані для кожного поясу окремо.

## На що впливає часовий пояс

### Розклад “День/Ніч”

Години **Початку дня** та **Початку ночі** використовуються в режимах, де прошивка прив’язується до часу доби (наприклад, нічна яскравість або нічні обмеження для звуку/дисплея).

Дивіться: [Загальні](../web-interface/sections/general.md)

### “Хвилина мовчання”

Режим “Хвилина мовчання” спрацьовує за локальним часом (о 09:00), тому часовий пояс напряму впливає на момент активації.

Дивіться: [Хвилина мовчання](minute-of-silence.md)

### Годинник на дисплеї

Якщо ввімкнено режим годинника, відображення часу також залежить від обраного часового поясу.

Дивіться: [Дисплей](../web-interface/sections/display.md)

## Що потрібно, щоб час був правильним

- пристрій має мати доступ до інтернету для синхронізації часу через NTP
- має бути заданий NTP сервер (за потреби)

Дивіться: [Мережа → NTP](../web-interface/sections/network.md)
